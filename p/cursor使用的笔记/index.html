Cursor的使用和介绍

参考的视频

1 下载安装

  • 直接去官网下载即可，

  • 前面自己选编辑器和Language即可，其他默认

  • 点第二个

  • 后面你可以直接导入VScode的客户端即可

  • 这里最好选隐私模式

2 使用Cursor

2.1 最核心的就是tab自动补全

3 Chat 和 Composer

3.1 怎么使用

  • 热键（Ctrl+I 和 Ctrl+L )如果是行就C^K

    • 我们可以选中一段代码后直接按着问

  • 两者类似，前者就是普通对待对话，后者则可以根据工作流来回答

  • 建议使用侧边栏问答，这样有上下文

3.2 选择模型

你可以在Setting-Models进行设置

  • 建议Chat功能选择gpt的模型
  • Composer的话建议使用claude
  • 注意模型的账号限量

Composer还区分Normal和Agent

  • 前者检测代码库和文档，还可以正常的创建和写入文件
  • 后者还可以执行命令，提取上下文等，更高级，但是只能用claude

3.3 具体案例

  • 我们打开哪个文件夹，那么这个文件夹就会被认为是一个项目

  • 使用@符号，可以选中某个想要的文件或者文件夹

  • 或者可以直接拖拽左侧文件过去/也可以直接用@来选中一块代码

3.4 添加文档

@doc可以添加本地文档，当然他默认会带一些，但是我们也可以自己添加；

后面带有offical的就是官方文档

  • Setting-Features-Docs中我们可以直接输入一个在线链接的地址
  • 注意如果只是一个单独的url的话，他就是检索当前页面
  • 否则就是检索当前页面和其子页面
  • 如果我临时有链接，可以直接@link+你的链接的话，他就会解析你的链接，否则他会把他当做prompt去看待

3.5 链接网页

使用@web就是可以直接让这个东西联网查找

3.6 链接git

@git可以链接某次提交，这样会很友好

尤其是在协作开发的时候，可以让ai看我们做了哪些更改/比较多个提交的差异

3.7 git提交自动生成commit

  • 如果你前面有比较规范的commit书写的方法的话，这里他可以自动帮你生成比较好的新的commit提交信息

3.8 notepad

  • 这个东西可以做一个临时的记事本

  • 注意Chat和Composer是不互通的，可以利用这个来做一个解决(存储对话记录)

  • 也可以用来记录一些待办事项

  • 如果需要新建的话，在右边做新增

3.9 codebase

  • @codebase是视频里面最推崇的做法
  • 他会这么做
  1. AI会根据我们的提示词，查找与指令相关的文件或者是代码块
  2. 重新排序：然后对收集到的代码排序，越相关的越靠前
  3. 推理
  4. 然后最后生成回应

虽然理论每次打开Cursor都会采集一次，但是为了以防万一，还是按照下面的路径来重新采集一次

  • 理论他会把你的数据上传到OpenAI上（应该是这个意思）做一个远程数据库检测
  • 他会检查gitignore，这样就可以防止秘钥等问题上传泄漏的问题
  • 如果没有，你也可以新建.cursorignore来做忽略，这样就可以减少被索引的文件信息量，从而提高检索效率/避免数据泄露

  • 理论来说，日志文件、环境变量（隐私数据）、构建文件、缓存信息文件、第三方信息配置文件都可以忽略
  • 你也可以让Cursor来帮你生成对应的文件

3.10 Cursor Rules

  • 这个是对Cursor的全局回复的设置

  • 对于某一个项目文件，我们可以生成一个cursorrules的文件只针对这个项目的规则

4 技巧和思路

4.1 检测

  • 你可以一个个查看来看是否修改

  • 我们也可以先点击save all然后运行，测试后在接收即可

4.2 回滚

Cursor会对每次回答做一个Checkpoint，我们可以做回滚、甚至重新编辑问题

4.3 预防（最泛化的）

4.3.1 让AI重复你的要求

在给AI的需求的时候，我们要先让他复述一遍需求，然后再答复操作，这样可以做二次核对

4.3.2 缩小指令的辐射范围

4.3.3 需求拆解

  • 如果需求不方便自己拆开给ai做，我们可以按照有序或者无序列表的形式来给到AI

4.3.4 把ai当做小孩子

  • 要做引导，尽可能清晰有逻辑，必要的时候给参考/自己的思路

4.4 如果我们对想要完成的不够清楚怎么办？

  • 应该先去chat模式去交流你的想法，向ai描述你的需求，然后问ai如果你想做怎么办？
  • 比如我想做一个网站，应该在chat问：
    • 我想做什么样子的
    • 有什么主流的技术，各自有什么优缺点
    • 我们应该怎么选择
    • 做完之后怎么发布
  • 然后再去composer里面去做

我们也可以在Chat里面问清楚之后，使用Notepad功能把问题拆解，然后一步步的喂给ai，得到方案（或者composer生成的有问题的话）可以再返回来去修改

4.5 使用Cursor的具体开发流程

如果你是程序员的话，希望Cursor帮你完成后续项目：

  • 你应该先让Cursor检索的你的项目，去做一个cursorignore文件

  • 然后做Codebase indexing下的Resvnc Index重新索引

  • 然后把所有的技术文档都录入到doc中以作备用

  • 然后再设置一个cursorrules文件，如果你不知道怎么写，可以问ai（最好是一条条写）

  • 如果后面写的东西有bug了，可以使用@git来看之前没有bug的版本，或者对于比较简单直接让composer做修改即可

  • 如果比较复杂就创建一个notepad，让他基于这个notepade里面的内容来例假问题

  • 这样以后这些notepad可以导出之后生成一个类似的doc文档，如果有一天再打开文档的时候，可以让ai快速理解

  • 当然记得及时归档，因为notepad只是Cursor的缓存

5 白嫖

  • 因为Cursor是有时间/API调用模型的限制的，到期之后只用不断的换邮箱注册即可

  • 我们可以使用一些临时邮箱

  • 无限邮箱

使用2925无限邮箱，实际上这个东西的话就是一个主邮箱，然后主邮箱后面跟一堆自由项的序号就行
